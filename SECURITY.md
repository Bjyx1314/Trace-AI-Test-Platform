# 安全政策

## 支持范围

仅当前主分支接受安全修复。早期版本可能发生不兼容变更。

## 报告漏洞

请通过代码托管平台的私密安全报告功能联系维护者，不要创建公开 Issue。报告应包含受影响版本、复现步骤、影响范围和建议缓解措施。维护者确认前请勿公开利用代码。

## 部署基线

- 默认首次登录凭据为 `admin` / `admin`；接入共享网络或生产环境前必须修改管理员密码。
- 替换其他所有示例密码和 `JWT_SECRET`，生产密钥至少使用 32 个随机字符。
- 仅通过 HTTPS 暴露平台，并限制数据库、Redis、Sonic 和 worker 接口的网络范围。
- 不把 `.env`、Playwright 登录态、上传文件、APK、备份或数据库卷纳入版本控制。
- 为外部 AI、飞书、SSO、制品库使用最小权限凭据并定期轮换。
- 生产环境关闭 mock，升级前备份 PostgreSQL，并审查第三方依赖漏洞。
